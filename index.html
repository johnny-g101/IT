Project Idea

Overview:

My proposed idea is to implement a chip that would be plugged in to an EV and ‘tune’ them from a user interface on a mobile deice such as a phone. This would mean that, in essence, the user can tweak and customise their EV from their phone. The chip would allow for unwanted features to be deactivated that usually would not be allowed to along with performance enhancements and the way the car drives can be changed at the touch of a button.

Motivation:

Personally, I have a lot of gripes with modern vehicles. The biggest one is all the electronic gismos and gadgets which I find incredibly annoying and distracting. I want feedback to DRIVE the car, not some computer to get it over and done with like a ‘chore’[4]. I would love an EV as a daily driver, but I don’t want the blandness that I would feel driving them. For example, one of my biggest criticisms that applies to most modern cars is the lack of steering feedback through the steering wheel because of the steering system that is utilised. This disconnected feel provides a lack of understanding as to what the car is doing through bends. Because the manufacturers of modern cars, EVs included, use this steering system as it is easier to drive, this reinforces a view that manufacturers OEM settings are to get driving ‘over and done with’[4]. My motivations is to improve the EV by providing a way for the spirited driver to tune their EV the way that they would like it. Thus, this may help the implementation of electric cars in Australia.

Description and Projection of idea:

The disabling of unwanted safety and enhancement features can be done by the user through their smart device. Some of my personal top picks include:

  • Lane Keep Assist
  • Radar Cruise control
  • Autopilot
  • Stability Control
  • Traction control
  • Blind Spot Monitoring
  • Speed Warnings
  • collision warning systems

It is thought that the user will select corresponding buttons on their smart device which represents each of these systems. When a button is pressed, this message is communicated to the hardware plugged into the EV. This hardware will modify or disable the code within the EVs onboard Computers to stop these features from operating.

It is also thought that the information screen on the dashboard (like in Tesla’s) could be reduced to absolute fundamental operations that the user wants. This may be able to be chosen in the app by the user. For example, if the user wants only Bluetooth connectivity for music and phone calls, and climate controls, this can be selected in some sort of click box fashion. Then the desired features would be the only thing that can be operated on the screen. There may even be a feature for the user to turn the screen off altogether if they so desire.

To enhance the users’ driving pleasure, the user will be able to activate performance/driver-feedback to enhance the driving experience of their EV. Instead of hydraulic power steering systems of old, modern cars, including EVs, mostly utilises the Electric Power Steering System (EPAS), which utilises a computer, an electric motor and torque sensors to apply assistance power to the steering column via a ball screw mechanism, which allows the motor to apply torque to the steering column whilst also allowing for slipping. This means that the motor will passively assist the steering column in the direction that the torque sensors detect the steering wheel to be spinning instead of directly spinning the column on its own. [5] [6]

Whilst EPAS systems require ‘less muscle’ to operate, this results in a ‘reduced steering feel’ through the steering wheel as this system has a ‘muted’ ‘mechanical connection’ compared to a hydraulic steering system [5]. The application should allow the user to control how much EPAS assistance that they want, with the option to turn it off altogether. The application will contain a slider, which will communicate to the hardware connected to the EV the amount of steering assistance that is to be provided by the motor. This should result in an enhanced mechanical feel provided by the steering system when taking a corner.

The user may also be able to enhance and change power output and curves provided by the electric motors. It is known that electric cars provide instant torque. This means that the maximum power of electric motors is gradually eased in when full throttle is applied to preserve mechanical and electrical components. However, the user may be able to select a function under a “performance options” menu, which allows instant full power as soon as full throttle is applied. This would provide enhanced throttle response, allowing for more driving pleasure.

Tuning and turning up power in an electric car could be paralleled with turning up the boost in a turbocharged Internal combustion engine powered car. However, instead of air pressure, it would be more like overclocking a computer; providing more power. The user would have the option to increase the charge taken (amp-hours) from the batteries by the motor controller to provide an increased charge to the motor. This would provide increased enjoyment and fun for the user.

Tools, Technologies and Skills Required:

First, the hardware will need to be designed by a skilled person who knows how to design electronic components so that it can be manufactured and coded to connect to EVs. This manufacturing process after final design is produced requires breadboard manufacturing, chip manufacturing and other electronic component manufacturing. It would then be required that the hardware is soldered together and code uploaded (which is a problem in itself).

Another technology problem to solve would be getting the device to read and modify the data for the many different makes of EVs. This is already a problem with Internal Combustion Engined vehicles as the ECUs and other modules are vehicular, let alone manufacturer specific. A majority of modern EVs utilise more complex computer systems, which may result in more variability between vehicles and manufacturers. It is thought that if one EV could be cracked and controlled, we might require the long and tedious process of getting EVs and writing alternative software to crack and operate that specific EV. If this option works, the mobile device might prompt this hardware to download these different files upon setup.

Another issue surrounding the operation of this product with EVs is potential software locks implemented by the manufacturer to prevent tampering. Manufacturers would be particularly motivated to prevent tweaks due to warranty, safety and maybe even paywall reasons. For example, Tesla did this to stop people bypassing paywalls and accessing certain reserved hardware for paying customers without paying [7]. This would require people that are good at finding exploits and loopholes in software to get around the EV manufacturers. It could be entirely possible that some makes, and models might not be able to be bypassed.

If this were to be commercially available, the liability of users either mechanically ruining their EVs through improper tuning or crashing them because of disabling safety features may become a problem. Particularly, crashing or battery fires because of overstress, potentially resulting in a serious injury or worse. A lawyer would be required to write a waiver when purchasing the product so that we are not responsible for anything. This is a better option then locking features as I am just as bad as the automakers then.

It also needs to be considered how the mobile device is going to wirelessly connect to the hardware. Bluetooth is too vulnerable for a system this important to someone’s vehicle. It might be required to use a local wi-fi network provided by the hardware which the device connects to, like a wireless printer. This then inconveniences the user because they no longer have internet to listen to music or use maps to direct where they are going
